# \## Part F: Daily Workflow with GitHub Desktop

# 

# \### For Everyone (Quick Reference):

# 

# \*\*Before starting work each day:\*\*

# 1\. Open GitHub Desktop

# 2\. Make sure you're on the right repository (`chess-game-dsa`)

# 3\. Click \*\*"Fetch origin"\*\* (top button) - this checks for updates

# 4\. If there are updates, click \*\*"Pull origin"\*\* to download them

# 

# \*\*After making changes:\*\*

# 1\. GitHub Desktop will automatically show your changes

# 2\. Review the changes in the left panel

# 3\. Write a summary (e.g., "Implemented Piece class")

# 4\. Click \*\*"Commit to main"\*\*

# 5\. Click \*\*"Push origin"\*\* to upload to GitHub

# 

# \*\*Creating a feature branch (for later):\*\*

# 1\. Click \*\*"Current Branch: main"\*\* at the top

# 2\. Click \*\*"New Branch"\*\*

# 3\. Name it (e.g., `feature/core-board-logic`)

# 1\. Click \*\*"Current Branch"\*\* dropdown
